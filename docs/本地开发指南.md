## 5. 组件 API

### 5.1 MarkdownRenderer Props

| 属性 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `content` | `string` | `""` | Markdown 内容 |
| `theme` | `ThemeType` | `"notion"` | 主题名称 |
| `view` | `"mobile" \| "desktop"` | `"mobile"` | 视图模式 |
| `showSettings` | `boolean` | `true` | 是否显示设置栏 |
| `onCopy` | `() => void` | `-` | 复制成功回调 |

### 5.2 ThemeType 类型

```typescript
type ThemeType = 'notion' | 'github-dark' | 'sakura' | 'sunset' | 'forest';
```

### 5.3 独立导出组件

```jsx
import { 
  MarkdownRenderer, 
  PreviewArea, 
  SettingsBar, 
  ThemeSwitch,
  useTheme,
  themes 
} from '@md-wx/react';

// 使用自定义设置栏位置
function CustomLayout() {
  const { theme, setTheme } = useTheme();
  
  return (
    <div>
      {/* 自定义位置的设置栏 */}
      <div style={{ position: 'fixed', top: 20, right: 20 }}>
        <ThemeSwitch theme={theme} onChange={setTheme} />
      </div>
      
      {/* 预览区域 */}
      <PreviewArea>
        <MarkdownRenderer content="# Hello" />
      </PreviewArea>
    </div>
  );
}
```

## 7. 常见问题

### Q1: npm link 后组件更新不生效？

**解决方案**：
```bash
# 在测试项目中重新链接
npm unlink @md-wx/react
npm link @md-wx/react
```

### Q2: 主题切换不生效？

**解决方案**：
- 检查主题名称是否正确
- 检查组件是否正确引入了主题样式
- 检查 CSS 变量是否正确定义

### Q3: 复制内容在微信公众号中样式丢失？

**解决方案**：
- 确保使用内联样式
- 避免使用 CSS 伪类和伪元素
- 避免使用复杂的 CSS 选择器
