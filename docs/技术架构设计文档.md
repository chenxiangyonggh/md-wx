### 3.2 模块划分

| 模块 | 职责 |
| :--- | :--- |
| **MarkdownRenderer** | 主组件，整合所有功能，接收 Markdown 内容，提供预览和交互 |
| **SettingsBar** | 设置按钮区域，包含主题切换、视图切换、复制按钮 |
| **PreviewArea** | 预览渲染区域，渲染 Markdown 内容，支持响应式 |
| **MarkdownParser** | Markdown 解析器，将 Markdown 转为 HTML |
| **StyleProcessor** | 样式处理器，处理微信兼容样式内联 |
| **CopyModule** | 复制模块，处理复制功能，转换为微信格式 |
| **ThemeSystem** | 主题系统，管理主题配置和切换 |

## 5. 关键设计

### 5.1 微信公众号兼容性设计

#### 5.1.1 微信公众号 HTML/CSS 规范要点

| 规范类型 | 具体要求 | 设计应对 |
| :--- | :--- | :--- |
| 样式内联 | 所有样式必须内联到标签 | StyleProcessor 自动将 CSS 转为 style 属性 |
| 标签限制 | 仅支持有限标签（p, h1-h6, code, pre, img, table 等） | 过滤不支持的标签，使用支持的标签替代 |
| 容器限制 | div 会被转成 p 或丢弃 | 使用 table 或 span 替代 div 布局 |
| 宽度限制 | 内容区最大宽度 677px | 预览区域固定宽度 677px |
| 颜色格式 | 使用十六进制颜色 | 主题颜色统一使用十六进制 |
| 代码块 | pre/code 可用，但超长行可能被强制换行 | 代码块添加横向滚动 |

## 6. 编码规范

### 6.1 文件命名规范

| 文件类型 | 命名规则 |
| :--- | :--- |
| 组件文件 | PascalCase |
| 工具函数 | camelCase |
| 类型定义 | PascalCase |
| 主题配置 | kebab-case |

### 6.2 代码风格规范

| 规范项 | 要求 |
| :--- | :--- |
| 缩进 | 2 空格 |
| 引号 | 单引号（字符串），反引号（模板字符串） |
| 分号 | 必须添加 |
| 空格 | 运算符前后加空格，大括号前加空格 |
| 函数 | 使用箭头函数（除非需要 this） |
| 组件 | 使用函数组件 + Hooks |

### 6.3 TypeScript 规范

| 规范项 | 要求 |
| :--- | :--- |
| 类型定义 | 所有函数参数和返回值必须有类型 |
| 接口命名 | PascalCase，以 I 开头 |
| 类型别名 | PascalCase |

### 6.4 CSS 规范

| 规范项 | 要求 |
| :--- | :--- |
| 类名 | 使用 TailwindCSS 类名 |
| 自定义样式 | 使用 CSS Modules 或内联样式 |
| 颜色 | 使用十六进制格式 |
| 单位 | 使用 px（微信兼容） |

## 9. 微信公众号样式兼容要点

### 9.1 支持的 HTML 标签

| 标签 | 用途 |
| :--- | :--- |
| `<p>` | 段落 |
| `<h1>`~`<h6>` | 标题 |
| `<code>` | 行内代码 |
| `<pre>` | 代码块 |
| `<img>` | 图片（必须使用 HTTPS） |
| `<table>` | 表格 |
| `<a>` | 链接 |
| `<strong>` | 粗体 |
| `<em>` | 斜体 |
| `<blockquote>` | 引用 |
| `<ul>`, `<ol>`, `<li>` | 列表 |
| `<hr>` | 分割线 |

### 9.2 支持的 CSS 属性

font-family、font-size、font-weight、color、background、text-align、line-height、margin、padding、border、width、height

### 9.3 注意事项

1. **样式必须内联**：微信不支持外部样式表和 style 标签
2. **避免使用 div**：div 会被自动转成 p 或丢弃
3. **颜色使用十六进制**：不支持 rgba 等其他格式
4. **图片必须 HTTPS**：HTTP 图片可能无法显示
5. **内容宽度不超过 677px**：避免横向滚动
6. **代码块添加背景色**：使用等宽字体
