## 2. 设计原则

### 2.1 核心原则

| 原则 | 说明 |
| :--- | :--- |
| **一致性** | 保持视觉元素、交互模式和布局结构的一致性 |
| **易用性** | 简洁直观的操作，降低用户学习成本 |
| **美观性** | 现代简约的视觉风格，符合当下设计趋势 |
| **响应式** | 适配不同屏幕尺寸和设备类型 |
| **微信兼容** | 复制内容完美适配微信公众号编辑器 |

### 2.2 设计理念

- **玻璃拟态**：顶部设置栏采用毛玻璃效果，现代感十足
- **卡片式布局**：预览区域采用圆角卡片设计
- **平滑过渡**：主题切换、视图切换均有流畅动画效果
- **清晰的视觉层级**：通过颜色、大小、间距建立清晰的信息层级

## 4. 组件设计规范

| 定位 | 固定定位（position: fixed） |

| 高度 | 自适应 |

| 圆角 | 12px |
| 阴影 | 标准阴影 |
| 最小宽度 | 180px |
| 选项高度 | 44px |

## 8. 微信公众号兼容性规范

### 8.1 支持的 HTML 标签

| 标签 | 用途 |
| :--- | :--- |
| `<p>` | 段落 |
| `<h1>`~`<h6>` | 标题 |
| `<code>` | 行内代码 |
| `<pre>` | 代码块 |
| `<img>` | 图片（必须 HTTPS） |
| `<table>` | 表格 |
| `<a>` | 链接 |
| `<strong>` | 粗体 |
| `<em>` | 斜体 |
| `<blockquote>` | 引用 |
| `<ul>`, `<ol>`, `<li>` | 列表 |
| `<hr>` | 分割线 |

### 8.2 样式规范

| 规范 | 说明 |
| :--- | :--- |
| 样式内联 | 所有样式必须内联到标签 |
| 颜色格式 | 使用十六进制（#RRGGBB） |
| 宽度限制 | 内容区最大 677px |
| 字体 | 推荐微软雅黑 |
| 图片 | 必须使用 HTTPS |

### 8.3 注意事项

1. **避免 div 标签**：微信会将 div 转为 p 或丢弃
2. **避免外部样式**：不支持 style 标签和外部样式表
3. **避免 JavaScript**：不支持 script 标签
4. **图片尺寸**：宽度不超过 677px
